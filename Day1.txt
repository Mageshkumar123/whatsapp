Cucumber:
---------
-->BDD(Behaviour Driven Development with the execution of TDD(Test Driven Development).
-->Used to convert the exact requirements into plain English Language using Gherkin Keywords.
-->It supports insprint automation.
-->Non-Technical people can also understand the automation coverage.
-->Cucumber supports Ruby,Java,Grovy,Python,.NET and PHP.
-->It provides different reports like xml,json and jvm reports.

BDD Framework:
--------------
-->Cucumber
-->Jbehave
-->Nbehave
-->Specflow

Cucumber Annotations:
---------------------
-->@Given
-->@When
-->@Then
-->@And
-->@CucumberOptions

Feature:(Test Scenario)
--------
-->Feature defines what we are going to test in the particular requirement/userstory.
-->Feature keyword is present at the starting of the feature file.

Scenario(Test Case)
---------
-->Scenario is how we are going to test the feature.

Given:
------
-->Some precondition step.

When:
-----
-->Some key actions.

And:
----
-->Used to add conditions in feature file

Then:
-----
-->To verify or validate something

Scenario Outline:
------------------
-->Used to execute the same scenario with multiple times wih different set of test datas.

Examples:
----------
-->Used to pass the parameters/datas using the | (pipe symbol) | separate the each data.

Background:
-----------
-->Define steps which is are common to all the scenarios in the feature file.
